Form Elements

Labels

There are 2 label modifier classes that you have to add besides the nc-input-group__label class to handle different background colors.
nc-input-group__label--dark
nc-input-group__label--light

Text fields

Example of an the initial state of the input text-field.

<div class="input-group">
<label for="email" class="input-group__label">
EMAIL<span class="input-group__required">*</span>
</label>
<input id="email" name="email" type="text" class="input-group__text-field">
</div>

Text fields in error state should add the --error modifier class to all the nucleus elements.

 

<div class="nc-input-group nc-input-group--error">
<label for="email" class="nc-input-group__label nc-input-group__label--error">
EMAIL<span class="nc-input-group__required nc-input-group__required--error">*</span>
</label>
<input id="email" name="email" type="text" class="nc-input-group__text-field nc-input-group__text-field--error" placeholder="EMAIL">
</div>

You can also use the nc-input-group__error-message class to show an error message.

This is a sampe error message to show when something goes wrong with the input field.

Drop downs

This is how we are going to implement drop downs.

<div class="nc-input-group dropdown">
<label for="state" class="nc-input-group__label">
STATE<span class="nc-input-group__required">*</span>
</label>
<select id="state" class="nc-dropdown__select">
<option value=""></option>
<option value="AL">Alabama</option>
<option value="AK">Alaska</option>
<option value="AZ">Arizona</option>
<option value="AR">Arkansas</option>
<option value="CA">California</option>
</select>
<svg class="nc-dropdown__icon-chevron">
<use xlink:href="img/sprite.svg#icon-chevron">
</svg>
</div>

Checkboxes

We use custom checkboxes that have a 1px border when unchecked. In the checked state, a white checkmark icon is seen against the $nc-brand-primary background with no stroke around the checkbox.
