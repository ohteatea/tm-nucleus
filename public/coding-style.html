Coding Style

This is a quick document outlining some of the SASS coding guidelines that are different from the tmol-web-spring project. A prettier version of this will come in the future. :)

  1. No usages of @extend.

  2. Do not use the parent selector shorthand (&) to compose class names.

    // bad
.parent {
  ...

  &__child {
    ...
  }
}

    Instead, explicitly write out any class names.

    // good
.parent {
  ...
}

.parent__child {
  ...
}

    The one exception is for pseudo-selectors.

    // this is okay
.link {
  ...

  &:hover {
    ...
  }
}

  3. No blank lines to separate stanzas anymore. Property declaration is still enforced, however.
